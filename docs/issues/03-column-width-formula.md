# Issue #03: CatalogStyle.cs 列宽公式解析

## 背景
原先每列宽度定义依赖对话框逐一配置输入，操作极其繁琐。

## 目标
支持传入类似 `"16+31+65+20"` 的合并连写字符串，后台一键自动解算各列绝对宽度。

## 具体任务

### 3.1 新增方法
```csharp
public void ApplyFormulaWidths(string formulaExpress)
```

### 3.2 解析逻辑
1. 使用 `+` 作为分隔符拆分字符串表达式（如 `"16+31+65+20"` → `["16","31","65","20"]`）
2. 遍历数组，安全转换为 `double` 类型
3. 解析失败时赋予默认值（如 30.0）

### 3.3 赋值逻辑
1. 按照索引顺序依次赋值给 `Columns` 属性列表中所有 `Visible = true` 的可见列
2. 如果解析出的数值数量少于可见列数量，其余可见列保持原宽度不变
3. 防止数组越界引发崩溃

### 3.4 使用示例
```
输入: "20+40+60"
可见列: [序号, 图号, 图名, 幅面]
结果: 序号=20, 图号=40, 图名=60, 幅面保持原宽度
```

## 验收标准
- [ ] 编译通过
- [ ] "16+31+65+20" 格式正确解析
- [ ] 解析失败项使用默认值 30.0
- [ ] 数值数量少于可见列时其余列保持原宽度
- [ ] 空字符串或 null 输入不崩溃
